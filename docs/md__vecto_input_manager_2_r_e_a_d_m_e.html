Vecto Input Manager (vectoim)

A command-line tool for creating, extracting, converting, and manipulating vehicle job components for VECTO simulations.

Features

  • Component Extraction: Decompose vehicle XML files into individual component files
  • XML <-> JSON Conversion: Convert between XML and JSON formats with schema validation
  • Standard Values Generation: Create certified components with standardized values
  • Vehicle Job Creation: Build new vehicle jobs by combining and replacing components
  • Multi-format Support: Work with both XML and JSON component files
  • Schema Validation: Built-in validation against VECTO schemas

Installation

Prerequisites

Ensure you have the .NET 9 Runtime installed on your system.

Download and Install

  1. Download the latest release and extract to a folder of your choice
  2. Install the .NET 9 Runtime from: https://dotnet.microsoft.com/en-us/download/dotnet/9.0/runtime

Platform-Specific Setup

Windows

Open Command Prompt or PowerShell in the installation folder and run:

vectoim --help

macOS / Linux

Open a terminal in the installation folder and make the executable:

chmod +x vectoim
./vectoim --help

Main Commands

vectoim provides four main commands:

Command Description
sv Create standard value components
job Build vehicle jobs from components
extract Extract components from vehicle XML files
json Convert XML files to JSON format

Help and Documentation

  • Get general help: vectoim --help
  • Get command-specific help: vectoim <command> --help
  • List available standard value components: vectoim sv --list
  • List available vehicles: vectoim job --list

Command Reference

sv - Create Standard Value Components

DESCRIPTION:
Create components with standard values
USAGE:
vectoim sv [component] [OPTIONS]
EXAMPLES:
vectoim sv battery -i battery.json -t -o battery.xml
ARGUMENTS:
[component] The component type to create. The default is 'auto' to
automatically detect the component type from the input JSON.
For a list of supported components, use argument --components
OPTIONS:
-h, --help Prints help information
-v, --verbose Enable verbose logging
-i, --input Input file containing the component data in JSON format
-t, --interactive Interactive Mode
-o, --output Output file for the generated XML or JSON
-m, --modify Values to set for the component. Use format: Key=Value,
e.g., R100CellsInSeries=20
-e, --example Generate an example JSON for the component
-l, --list List of available components

job - Create Vehicle Jobs

DESCRIPTION:
Create jobs with components
USAGE:
vectoim job [vehicle] [OPTIONS]
EXAMPLES:
vectoim job -i vehicle.xml -i component_*.xml -o new_vehicle.xml
ARGUMENTS:
[vehicle] The vehicle type to create
OPTIONS:
DEFAULT
-h, --help Prints help information
-v, --verbose Enable verbose logging
-i, --input Input files for the job. Multiple files
or glob patterns can be specified
-t, --interactive none Interactive Mode
-o, --output Output file for the generated XML or JSON
-k, --key-value-file Generate a values file (.values.txt)
containing all keys and values assigned
during vehicle creation
-m, --modify Values to set for the job. Use format:
Path.Key=Value, e.g.,
Components.AxleWheels.Data.Axles[
0].TwinTyres=True
-e, --example Generate an example JSON for the
component
-l, --list List of available vehicles

extract - Extract Components from Vehicle

DESCRIPTION:
Extract components from vehicle
USAGE:
vectoim extract [OPTIONS]
EXAMPLES:
vectoim extract -i vehicle.xml -o ./folder
OPTIONS:
DEFAULT
-h, --help Prints help information
-v, --verbose Enable verbose logging
-i, --input Input vehicle XML or JSON to extract components
from
-o, --output Output directory where extracted component files
will be saved
-f, --format xml Output format for extracted components. Valid
options: xml, json, both. Default is xml

json - Convert XML to JSON

DESCRIPTION:
Convert XML vehicle and component files to JSON format
USAGE:
vectoim json [OPTIONS]
EXAMPLES:
vectoim json -i vehicle*.xml -o ./json --remove-components
OPTIONS:
-h, --help Prints help information
-v, --verbose Enable verbose logging
-i, --input Input XML files to convert. Multiple files or
glob patterns can be specified
-o, --output Output directory for the converted JSON files
-r, --remove-components Remove (nullify) all components from the vehicles
before converting to JSON

Usage Examples

1. Extract Components from a Vehicle

Extract all components from a vehicle XML file to JSON format:

vectoim extract -i Conventional_HeavyLorry.xml -o ./components -f json

Extract to both XML and JSON formats:

vectoim extract -i vehicle.xml -o ./output -f both

2. Convert XML to JSON

Convert a vehicle XML file to JSON:

vectoim json -i vehicle.xml -o ./json

Convert multiple files with wildcards:

vectoim json -i *.xml -o ./json_output

Remove components before converting:

vectoim json -i vehicle*.xml -o ./json --remove-components

3. Create Standard Value Components

List all available standard value component types:

vectoim sv --list

Generate a template for a battery component:

vectoim sv battery --example

Create a standard value component from a JSON input file:

vectoim sv battery -i battery.json -o battery.xml

4. Build New Vehicle Jobs

Replace components in an existing vehicle:

vectoim job -i original_vehicle.xml -i new_engine.xml -i new_battery.json -o modified_vehicle.xml

Use glob patterns to include multiple components:

vectoim job -i vehicle.xml -i "Components.*.xml" -o output.xml

File Naming Conventions

When extracting components, vectoim uses descriptive filenames that indicate:

  • Component type: Engine, Gearbox, Battery, etc.
  • Component path: Location within the vehicle structure
  • Array indices: For components that appear multiple times

Examples:

  • Components.Engine.json - Main engine component
  • Components.AxleWheels.Data.Axles[0].Tyre.json - First axle's tyre
  • Components.Powertrain[1].ElectricMachine.json - Second powertrain's electric machine

Logging

vectoim creates log files for each run to help with troubleshooting and debugging. Each execution generates a separate log file with a timestamp in the filename. You can also increase the verbosity of the output logs by including the -v parameter.

Log File Naming

Log files follow the pattern: vectoim-YYYY-MM-DD-HHMMSS.log

Example: vectoim-2025-10-25-143022.log

Windows location:

C:\Users<YourUsername>\AppData\Local\vecto\

To access:

  1. Press Win + R
  2. Type: LOCALAPPDATA%\vecto
  3. Press Enter

Linux location:

/home/<yourusername>/.local/share/vecto/

macOS location:

/Users/<yourusername>/Library/Application Support/vecto/

Logs from the last 100 runs are maintained and older logs are deleted.
